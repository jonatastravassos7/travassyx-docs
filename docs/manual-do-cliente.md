# Manual Público do Cliente

Este manual resume as principais áreas da Travassyx Platform para usuários, managers e administradores de cliente.

## Dashboard

Mostra a saúde geral do ambiente, disponibilidade, recursos, riscos, capacidade, alertas e indicadores operacionais.

Use para uma leitura rápida do estado do ambiente.

## Command Center

Centraliza decisões operacionais:

- incidentes;
- alertas;
- correlação NOC;
- evidências;
- causa provável;
- ações recomendadas;
- runbooks.

Use para priorizar o que precisa ser tratado primeiro.

## Copilot

Assistente contextual da plataforma. Ele responde dúvidas sobre telas, indicadores, integrações, incidentes, runbooks, FinOps e operação do ambiente selecionado.

## Serviços

Conecta recursos técnicos a serviços de negócio. Ajuda a entender impacto por aplicação, owner, criticidade, dependências e risco operacional.

## FinOps

Mostra custos, desperdícios, oportunidades de economia, budgets, forecast, tags e recomendações financeiras quando o módulo estiver habilitado.

## Inventário

Lista recursos monitorados, como VMs, hosts, datastores, instâncias cloud, volumes, buckets, contas e outros itens coletados.

O Recurso 360 abre o detalhe de um recurso com métricas, histórico, timeline, sinais NOC e contexto operacional.

## Histórico & Logs

Mostra eventos, coletas, alertas, timeline, qualidade dos dados e trilhas de investigação.

## Incidentes

Transforma alerta em tratativa:

- prioridade;
- owner;
- causa provável;
- evidência;
- runbook;
- comentários;
- resolução;
- histórico.

## Administração

Organiza operação e configuração:

- clientes ou ambientes;
- providers;
- cloud accounts;
- políticas e alertas;
- usuários;
- LDAP/AD;
- SMTP;
- backup;
- licença;
- documentos legais;
- integrações enterprise;
- atualizações da plataforma.

## Perfil do Usuário

Permite ver sessão, conta, MFA, senha e documentos legais aceitos.

## Boas Práticas

- Use Dashboard para leitura rápida.
- Use Command Center para decisão.
- Use Recurso 360 para investigação técnica.
- Use Incidentes para registrar tratativa.
- Use Copilot para orientação.
- Mantenha cadastro do ambiente atualizado.
- Revise backup antes de atualização.
- Não feche incidente sem causa, ação e validação.
