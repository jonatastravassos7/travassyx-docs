# Segurança

Se você acredita ter encontrado uma vulnerabilidade na Travassyx Platform, reporte de forma responsável pelos canais oficiais de suporte ou contato comercial.

## O Que Enviar

Inclua:

- descrição do problema;
- impacto esperado;
- passos de reprodução;
- versão da plataforma;
- modo usado: SaaS, instância ou Community;
- evidências sem expor dados sensíveis de terceiros.

## O Que Não Fazer

- Não publique exploração antes de resposta da Travassyx.
- Não acesse dados de terceiros.
- Não execute testes destrutivos em ambientes de produção.
- Não compartilhe serial, token, senha, chave privada ou credencial em issue pública.

## Boas Práticas para Clientes

- Use HTTPS.
- Proteja credenciais e seriais.
- Mantenha backup habilitado.
- Configure MFA para administradores.
- Use LDAP/AD ou SSO quando disponível.
- Restrinja acesso administrativo.
- Atualize a plataforma conforme releases oficiais.
